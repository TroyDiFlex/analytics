Логин GitHub: TroyDiFlex

## Что в итоге получится

Сайт с твоим роадмапом обучения:
- Заходишь по ссылке с любого устройства
- Первый раз вводишь секретный ключ — больше не спрашивает
- Ставишь галочки, пишешь заметки, выполняешь задания
- Прогресс сохраняется в Google Таблице — только ты имеешь доступ

---

## Архитектура (как всё устроено)

```
Публичный репо на GitHub (TroyDiFlex)
— код сайта виден всем, тебе пофиг
         ↓
GitHub Pages — бесплатный хостинг
         ↓
Ссылка вида: troydiflex.github.io/analytics
         ↓
При первом открытии — вводишь секретный ключ
Сохраняется в твоём браузере, больше не спрашивает
         ↓
Сайт при каждом запросе отправляет этот ключ в Google
         ↓
Apps Script проверяет ключ:
  — правильный → читает/пишет данные в твою таблицу
  — чужой      → отказ, данные не отдаёт
```

---

## Что где хранится и кто видит

| Что | Где | Кто видит |
|---|---|---|
| Код сайта | Публичный GitHub | Все (тебе пофиг) |
| Секретный ключ | Только в твоём браузере | Только ты |
| Данные (галочки, заметки, даты) | Google Таблица | Только ты |
| Проверка ключа | Apps Script в Google | Только ты |

**Сторонних сервисов — ноль. Платить — ничего.**

## Защита данных

Случайный человек заходит на сайт → видит форму ввода ключа → не знает ключ → данных не видит, ничего не меняет, ничего не удаляет.

Даже если кто-то найдёт ссылку на Google Apps Script — без секретного ключа он получит только ошибку. Ключ нигде в публичном коде не прописан.

## Из чего состоит сайт

### Левая колонка — навигация
- Общий прогресс: «12 из 32 тем · день 14»
- Список всех 8 этапов с иконками и мини-прогрессом

### Правая колонка — содержание этапа
1. **Темы** с галочками (ставишь → записывается дата; снимаешь → дата удаляется)
2. **Подсказки** — жёлтые блоки, скрыты по умолчанию, раскрываются кликом
3. **Практические задания** с чекбоксами и полем для заметки
4. **Поле «Мои заметки»** к этапу — пишешь что угодно
5. **Список ресурсов**
6. **Кнопка «Сбросить прогресс этапа»** — с подтверждением

### Шапка
- Название + общий прогресс
- Кнопка «Скачать прогресс» — сохраняет всё в файл

## Практические задания по этапам

### Этап 1 — Погружение
- Открой hh.ru, найди 5 вакансий. Выпиши в заметку: какие 3 инструмента встречаются чаще всего?
- Запиши одним абзацем: чем продуктовый аналитик отличается от маркетолога и от дата-инженера?

### Этап 2 — Excel/Google Таблицы
- Скачай датасет Superstore с Kaggle. Отфильтруй строки с продажами >500$. Сколько таких строк?
- Создай сводную таблицу «сумма продаж по категориям». Какая категория лидирует? Добавь колонку «прибыль в % от продаж».
- Используй VLOOKUP или INDEX+MATCH: подтяни среднюю продажу по категории из справочника.

### Этап 3 — SQL
- На sqliteonline.com создай таблицу users (id, name, city, age), добавь 10 строк. Выбери пользователей старше 25 лет из Москвы.
- Посчитай количество пользователей по городам. Выведи только города, где их больше 2.
- Создай таблицу orders (id, user_id, amount). Напиши запрос: имя пользователя + общая сумма его заказов.
- Пронумеруй пользователей внутри каждого города по убыванию возраста через ROW_NUMBER().
- Реши 3 лёгких задачи на StrataScratch. Вставь скриншоты в заметку.

### Этап 4 — Метрики продукта
- По данным Superstore: посчитай количество уникальных покупателей по месяцам (аналог MAU). Какой месячный пик?
- Построй когортную таблицу retention: когорта = месяц первого заказа, столбцы = месяцы после него. Минимум 3 когорты.
- Кейс: «Retention упал с 40% до 25% за месяц. Запиши 5 гипотез почему и как бы ты каждую проверил».

### Этап 5 — Статистика
- На любом числовом поле датасета: посчитай среднее, медиану, p25, p75, p95. Есть ли разрыв между средним и медианой? Объясни почему.
- Открой Evan's A/B Testing Tools. Введи: контроль 5%, тест 6%, выборка 1000 человек. Значим ли результат? Попробуй 500 человек. Запиши вывод.

### Этап 6 — Визуализация
- В Tableau Public создай дашборд по Superstore: тренд продаж по месяцам + топ-10 товаров + карта по регионам. Опубликуй — вставь ссылку в заметку.
- Посмотри на свой дашборд и запиши 3 конкретных инсайта.

### Этап 7 — Python (необязательный)
- В Jupyter Notebook загрузи Superstore через pandas. Повтори свой GROUP BY запрос через groupby() — результат должен совпасть с SQL.
- Построй гистограмму поля Sales через matplotlib. Есть ли выбросы?

### Этап 8 — Портфолио
- Напиши 5 SQL-запросов по Superstore: топ-10 товаров по выручке, аутсайдеры по прибыли, средний чек по категориям, динамика по месяцам, покупатели с 2+ заказами.
- Оформи README.md: задача → данные → методы → 3 инсайта. Не больше 300 слов.
- Кейс: «В мобильном приложении банка DAU упал на 20% за неделю. Опиши свои действия пошагово».
- Реши 5 задач Medium уровня на LeetCode (SQL раздел).

## Хранение прогресса — Google Таблица

Прогресс хранится в твоей личной Google Таблице. Сайт общается с ней через Apps Script — маленькую программку внутри Google, которая проверяет секретный ключ перед каждой операцией.

### Что нужно сделать один раз (я напишу пошаговую инструкцию):
1. Создать новую Google Таблицу
2. Зайти: Расширения → Apps Script
3. Вставить код (дам отдельным файлом)
4. Нажать «Развернуть» → «Веб-приложение»
5. Скопировать полученную ссылку → вставить в настройки сайта
6. При первом открытии сайта ввести свой секретный ключ

## Ответы на частые вопросы

**Может кто-то чужой прочитать мои заметки?**
Нет. Без секретного ключа Apps Script не отдаёт данные.

**Может кто-то удалить мои данные или поставить галочки?**
Нет. Каждая операция проверяет ключ. Нет ключа — нет действия.

**Может кто-то увидеть мой секретный ключ в коде?**
Нет. Ключ нигде в коде не написан. Он хранится только в твоём браузере.

**Нужно ли что-то платить?**
Нет. GitHub бесплатно. Google Таблицы бесплатно. GitHub Pages бесплатно.

**Что если зайду с нового устройства?**
Введёшь ключ один раз — и он сохранится в браузере этого устройства.
